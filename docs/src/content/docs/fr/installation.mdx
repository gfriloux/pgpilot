PGPilot est un gestionnaire de clés graphique pour GPG. Ce guide couvre l'installation à partir des versions GitHub et la compilation à partir de la source.

## Conditions préalables

PGPilot nécessite :

- **GnuPG** ≥ 2.2 (la commande `gpg` doit être dans `PATH`)
- **gpg-agent** (généralement fourni avec GnuPG)
- **pinentry** — un programme pinentry pour les invites de mot de passe (ex: `pinentry-tty`, `pinentry-gtk2` ou `pinentry-qt`)

### Vérifier les prérequis (Linux)

```bash
gpg --version
# La sortie doit afficher GnuPG version ≥ 2.2

gpg-agent --version
# Confirmez que l'agent est disponible

which pinentry
# Devrait trouver un binaire pinentry (ou vérifier : gpg-conf --list-dirs | grep pinentry)
```

Si `gpg` est manquant :

- **Ubuntu / Debian** : `sudo apt install gpg gnupg-agent pinentry`
- **Fedora / RHEL** : `sudo dnf install gnupg pinentry`
- **Arch** : `sudo pacman -S gnupg pinentry`
- **NixOS** : Utilisez `nix-shell -p gnupg gpg-agent pinentry` ou ajoutez à `flake.nix`/`shell.nix`

## Installation à partir des versions GitHub

1. Visitez [versions PGPilot](https://github.com/gfriloux/pgpilot/releases)
2. Téléchargez le dernier binaire `pgpilot` pour votre distribution Linux (amd64, arm64, etc.)
3. Rendez-le exécutable :
   ```bash
   chmod +x pgpilot
   ```
4. Exécutez-le :
   ```bash
   ./pgpilot
   ```

Pour installer au niveau du système :

```bash
sudo mv pgpilot /usr/local/bin/
pgpilot  # fonctionne maintenant de n'importe où
```

## Compilation à partir de la source

### Exigences

- **Rust** ≥ 1.80 (installez à partir de [rustup.rs](https://rustup.rs) si nécessaire)
- **Nix** (optionnel mais recommandé sur NixOS) — voir [nix.dev](https://nix.dev)
- **clang** et **en-têtes de développement** pour le support Wayland/X11

### Cloner le dépôt

```bash
git clone https://github.com/gfriloux/pgpilot.git
cd pgpilot
```

### Compiler dans le shell dev Nix (recommandé)

Si vous avez `nix` installé :

```bash
nix develop
cargo build --release
```

Le shell dev configure `LIBCLANG_PATH` et `LD_LIBRARY_PATH` pour toutes les dépendances (sequoia-openpgp, iced/Wayland, dbus, etc.).

### Compiler sans Nix

```bash
cargo build --release
./target/release/pgpilot
```

**Remarque** : Sur les systèmes sans environnement préconfigré, vous devrez peut-être installer des packages supplémentaires (clang, libssl-dev, libdbus-1-dev, libwayland-dev, etc.). Consultez la documentation de votre distribution si la compilation échoue.

## Vérification

Une fois installé, vérifiez que PGPilot fonctionne :

1. Lancez PGPilot :
   ```bash
   pgpilot
   ```

2. Une fenêtre doit apparaître. Sinon, vérifiez que votre serveur d'affichage (X11 ou Wayland) est en cours d'exécution :
   ```bash
   echo $DISPLAY     # X11
   echo $WAYLAND_DISPLAY  # Wayland
   ```

3. Ouvrez la page **Diagnostic** (barre latérale, bouton "Diagnostic") :
   - Tous les contrôles doivent afficher « OK » ou « Info » (icônes vertes/bleues)
   - Si vous voyez « Erreur » en rouge, suivez le guide de dépannage (voir [Dépannage](/pgpilot/fr/troubleshooting/))

## Désinstallation

Pour supprimer PGPilot :

```bash
# Si installé via /usr/local/bin/
sudo rm /usr/local/bin/pgpilot

# Pour aussi supprimer la config et les données en cache :
rm ~/.config/pgpilot/config.yaml
```

**Remarque** : PGPilot ne modifie pas directement votre trousseau GPG — toutes les opérations déléguées au binaire `gpg`. Vos clés restent dans `~/.gnupg/` et sont sûres de être sauvegardées indépendamment.

---

## Supprimer les données du trousseau GnuPG

> **Avertissement : Ceci ne fait PAS partie de la désinstallation de PGPilot.**
>
> `~/.gnupg/` est le répertoire de données propre à GnuPG. Il contient **toutes vos clés GPG**, y compris les clés d'autres applications, les clés que vous avez importées au fil des ans et vos clés privées. La supprimer est **permanente et irréversible** à moins que vous ayez une sauvegarde.
>
> PGPilot ne possède ni ne modifie ce répertoire — il délègue toutes les opérations au binaire `gpg` qui lit depuis `~/.gnupg/`.

Ne procédez que si vous avez l'intention de **détruire complètement votre trousseau GPG** et avez vérifié vos sauvegardes.

**Conditions préalables avant de continuer :**

1. Exportez et stockez de manière sécurisée toutes les clés privées que vous souhaitez conserver
2. Sauvegardez vos certificats de révocation
3. Vérifiez que vos sauvegardes peuvent être restaurées

```bash
# IRRÉVERSIBLE — détruit toutes les clés GPG
rm -rf ~/.gnupg/
```
